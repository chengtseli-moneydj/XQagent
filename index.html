當 XQ 遇上 Agent 時代

把鍵盤操作員變回真正的金融專業人士
——從手動操作 UI 與 xlsx，到「可驗證、可對話」的決策系統提案

提案文件
1

被 UI 綁住的專業：當你被迫成為高級資料搬運工

The Problem: Professionals trapped by UI operations

如果把一位重度 XQ 使用者的一天攤開來看，你會發現一個有點殘酷的事實：真正需要專業判斷的時間，其實被大量的「機械工作」包圍著。

早上開機，先啟動 XQ，看大盤、開個股、切幾個自訂分頁。看到有興趣的標的，點進去看 K 線、籌碼、法說簡報。接著開 Excel，把關鍵數字抄進去，調整模型假設。下午要開會，又得從 XQ 截圖技術線圖、把籌碼表格匯出成 xlsx、貼到 PowerPoint，調顏色、改標題，順便再補幾張從券商系統抓來的持股截圖。

這一整套流程裡，真正只有你能做的，是「判斷」：哪家公司值得花時間研究？哪個財報訊號是真正的轉折？這個策略在當前市場環境還站得住腳嗎？

但你卻把大半精神浪費在：

  • 在 XQ 各個頁籤之間來回切換，只為了找到一個欄位。
  • 手動把數字搬到 Excel，再小心翼翼地檢查公式。
  • 為了讓簡報好看，調整每一個圖表和文字方塊的位置。

XQ 已經把行情、財報、籌碼、策略、下單整合在一起，算是台灣少數真正「一條龍」的金融工具。但我們使用它的方式，還是停留在「用滑鼠一格一格取用」，再自己用 Excel 把世界拼回來。

如果仔細問一句：在這些工作裡，哪些是「非我不可」的？你會發現，至少有一半以上，根本只是高級的資料搬運與格式美編。

這，正是 Agent 時代要解決的第一個問題：不是再多做幾個漂亮的 UI，而是想辦法把人從這些機械流程裡解放出來。

2

如果那些繁瑣的小事都有人代勞：工作會長成什麼樣？

The Vision: What if tedious tasks were handled for you?

不妨認真玩一個「what if」。

如果有一個 XQ Agent，永遠開在你工作環境裡，專門負責所有你不想再做第二次的瑣碎動作，你的一天會變得多不一樣？

用戶對 Agent 說

「幫我更新昨天追蹤的十檔股票：整理一下財報、重大新聞、籌碼變化，順便幫我看哪幾檔偏離原本假設最多，做一頁簡報給下午投資會議用。」

於是 Agent 自己去做你本來手動做的事：在 XQ 裡抓價量、開財報、看主題族群、跑你平常用的 XS 選股條件，甚至去比對你過去寫過的備忘錄假設，找出差異最大的地方。最後回來的不是一堆截圖，而是一份帶圖表的簡報草稿，每個數字都可以一鍵跳回 XQ 的原始來源。

你如果不放心，可以問得更細

「為什麼你覺得這一檔的風險變高了？幫我把你引用的兩份財報段落標出來，還有 EPS 的敏感度表重算一次給我看。」

Agent 不只是「講講而已」，而是能真的打開模型，把假設調整後重新計算，並把差異整理成好懂的圖和文字。你不是再去改公式，而是去看：「這組假設我相信嗎？如果不相信，要怎麼改？」

到了下午，你開著那份由 Agent 產生的報告，面對長官或客戶，不用邊講邊擔心有人問：「這個數字是從哪裡來的？你是怎麼算出來的？」

因為你只要再對 XQ Agent 說一句：「幫我把這頁 EPS 假設的來源與計算步驟展開。」它就會即時呈現計算路徑，甚至附上原始財報段落。在場的人若有不同看法，可以直接說：「那你用保守一點的假設重算一次，我們來看結果差多少。」

這時文件不再是「定版的 PDF」，而是一個可以互動、可以重跑情境的決策容器。

在這樣的世界裡，人類真正做的事情只剩下三件：

說出目標與限制

我要達成什麼、不能踩到什麼線

審查 Agent 的流程與假設

這樣想合理嗎、有沒有漏掉什麼

對最後的決策結果負責

要不要買、賣，或建議客戶加碼

工作分工的新原則

所有「可以教會別人做」的工作，都交給 Agent；唯有「必須由你親自扛」的部分，才是你真正的價值所在。

看世界在做什麼：Anthropic 與全球金融機構的合作樣貌

Global Practice: How leading institutions are adopting AI Agents

這樣的想像，並非憑空幻想，而是正在全球金融業發生的事。

Anthropic 在 2025 年推出了專門面向金融業的 Claude for Financial Services，把市場資料、內部資料平台（像 Databricks、Snowflake）與實時行情，整合成一個可被 Agent 操作的介面，讓金融專業人士可以在同一個環境裡查數據、建模型、寫報告，而且每一個結論都有可追溯的來源連結。

NBIM 挪威主權財富基金 1.7 兆美元

全球最大的主權基金之一，在全公司推廣 Claude，讓超過 600 位同仁都能用 Agent 來讀研究報告、查投資組合 9,000 家公司的 ESG 與財務資料、生成內部備忘錄與合規文件，平均每個人一週省下約 20% 的時間，累積相當於 20 多萬小時的效率提升。更重要的是，他們不是只拿 AI 來「寫摘要」，而是讓投資與風險團隊可以直接在 Claude 裡查詢自家 Snowflake 資料倉，對接內部投資數據與外部市場資訊，做出可稽核的分析流程。

對 XQ 的啟示：真正產生差異的，不是「有沒有用 AI」，而是「能不能安全地把內部投資資料倉變成 Agent 的工具層」。

LSEG 倫敦證交所集團 MCP 協定整合

將 Workspace 與金融分析工具的授權數據，透過與 Anthropic 的合作串接至 Claude，讓使用者可以在 Agent 裡存取專業金融資訊、摘要法說會、觸發工作流程，這些整合都是透過符合 MCP 協定的伺服器來完成。（MCP 是一個讓不同系統可以用統一方式被 Agent 呼叫的開放協定，對 XQ 而言，就像是「給 Agent 用的標準化 API 規格」。）

對 XQ 的啟示：XQ 可以扮演「台灣專業金融工具＋資料供應商」，透過類 MCP 方式，成為各家 Agent 的標準入口，而不只是一個終端 UI。

Claude for Excel 垂直優化

Anthropic 針對金融工作流程推出的產品能力，讓 Agent 可以直接讀懂、修改、建立試算表，同時維持公式結構，不只會「幫你填數字」，還能協助 debug、補齊模板、依照文字指令從零打造新的模型。

Chronograph MCP 整合

把私募股權的投資組合監控數據接進 Claude，讓私募基金可以在 Agent 裡查詢投資組合資料，透過開放協定實現數據互通。

換句話說，世界上已經有人把「AI 金融助手」從 demo 變成實際戰力：它連上真實的數據倉、真實的行情與風控系統，幫真正管錢的人省下大量時間，還留下完整、可查的分析紀錄。

對 XQ 而言，這不是一個遙不可及的故事，而是一面非常具體的鏡子：當全球最大的主權基金、專業數據供應商、投資科技公司都在用 Agent 重寫工作流程時，台灣這套已經深度紮根市場多年的核心工具，要選擇扮演什麼角色？

XQ 在 Agent 時代的產品調整藍圖：從畫面到工作流的四個轉向

Strategic Blueprint: Four key shifts for XQ's transformation

如果接受一個前提：「未來的使用者會越來越習慣用 Agent 來做事」，那 XQ 要變的，就絕對不只是多一個對話視窗，而是整體產品哲學的轉向。

可以從四個方向來想這個新的 XQ：

1

把「Agent」當成第一順位的使用者，而不是只服務「人眼」

現在的 XQ 功能，幾乎都是為了人類 UI 設計：每個畫面有自己的欄位、按鈕與查詢方式。在 Agent 時代，需要再多一層「給 Agent 用的工具層」：行情、財報、籌碼、策略結果、券商持股，都要有標準化的結構輸出與調用限制，讓 Agent 能夠像工程師一樣，自行決定查什麼、查多久、查多細。這不一定要對外開放成 API，哪怕只是在 XQ 內部做一層「工具目錄」，也能讓未來的 XQ Agent 有地方可以插手、而不必假裝自己在「幫使用者按按鈕」。

2

從「畫面導向」改成「工作流導向」

今天的 XQ，是一個功能集合：看盤是一個頁籤、選股是一個頁籤、XS 是一個頁籤、籌碼是一個頁籤。但使用者真正在做的，是一條條工作流：從「發現一檔標的 → 查資料 → 寫備忘錄 → 決定下單 → 事後檢討」。Agent 版 XQ 的任務，是把這些工作流變成可重複、可記錄的流程模版：當使用者第一次做完某條研究流程，XQ Agent 可以問：「要不要把這套流程存起來？」以後只要說「幫我照上次那套流程重跑一次，但換成其他同族群股票」，整個鏈條就能自動再走一遍。這樣一來，XQ 不再是讓你「每次都重新來過」的一堆工具，而是越用越懂你的智慧工作平台。

3

讓「文件」變成 XQ 的主場，而不是最後才被 export 的副產品

現在的決策輸出，幾乎都落在 Excel、PowerPoint 或 PDF，而且一旦匯出，就和 XQ 斷線。Agent 版 XQ 可以反其道而行：在 XQ 裡內建「備忘錄／建議書／會議報告」這些文件型態，讓 Agent 直接幫你生第一版。文件裡的每一個關鍵數字，都對應到某份報表、某張圖、某次回測；你要查來源，只要點一下；你要改假設，只要跟 Agent 說一聲，它就會在原始模型上重跑，文件也跟著更新。久而久之，XQ 會變成決策歷史的中樞：過去所有重要決策都有文件、有數據、有假設，未來也能拿來對照檢討。這會讓 XQ 在使用者心目中，從「看盤神器」變成「決策記憶體」。

4

用漸進式路線，把現有使用者帶進 Agent 世界

現實世界不能砍掉重練，所以導入順序很重要：

  • Phase 1：文件優先 — Agent 先幫忙產出研究摘要、投資備忘錄、客戶簡報，每個數字都連回 XQ。侵入性最低，效益最快被感受到。
  • Phase 2：研究流程 Agent 化 — Agent 介入選股條件設計、策略回測與情境分析，成為會用 XS、會讀籌碼、會跑模型的量化助理。
  • Phase 3：風控＋下單輔助 — 在嚴格權限與審核下，Agent 連上真實下單與風控環節，處理再平衡、風險報表、自動監控等任務。

在這整個過程中，XQ 真正要守住的，有兩件事：

可驗證

任何 Agent 產出的數字與結論，都要查得到來源、看得到過程

人負責

最終決策權仍在使用者手上，Agent 只是讓人有更完整的資訊與更清楚的假設

當這兩件事都被設計進產品裡時，XQ 不但不會被 Agent 時代淘汰，反而有機會成為台灣金融圈裡第一個真正「Agent 原生」的專業系統。
